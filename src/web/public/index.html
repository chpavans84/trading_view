✅ Trade Entry Checklist

Steps the bot runs on every scan (in order):
  1. Daily P&L → stop if target hit or loss limit reached
  2. Market status → skip if closed
  3. Positions → stop if at max open positions
  4. VIX → stop if above max-VIX setting
  5. Build candidate list (movers + earnings + sectors)
  6. Filter to 4–6 best candidates
  7. Run conviction score on each → skip if below threshold
  8. Execute highest-scoring candidate

🔍 How the Scanner Finds and Executes Trades
Every 10 minutes during market hours this 7-step pipeline runs automatically — or manually from the Scanner tab. Here's exactly what happens inside each step.
1
Trigger
Auto: node-cron fires every 10 min, 9:30am–3:50pm ET, weekdays only
Manual: "▶ Run Scan" button in Admin → Scanner tab (analysis only)
Auto-trade: "▶ Run & Trade" — executes if selection qualifies
6 total background jobs: scan (10min) · position monitor (2min) · morning briefing (9:30am) · EOD summary (4pm) · stale position close (3:50pm) · regime refresh (15min)
node-cron Manual UI trigger
2
Pre-flight gate checks — abort if any fail
All gates must pass or scan stops here:
☑ getMarketStatus() — market must be OPEN
☑ Daily P&L within limits — not hit $150 profit target or -$200 loss limit
☑ VIX below max_vix (default 30) — above this is too dangerous
☑ Open positions below max_open_positions (default 2)
☑ Time: 9:45am – 3:15pm ET only (no entries outside these hours)
☑ Scanner not manually paused by admin
Hard stops — abort on any fail PostgreSQL scanner_state
3
getMarketContext() — understand the battlefield (15-min cache)
Runs 4 sub-fetches in parallel via Promise.allSettled():
  • getMarketSentiment() → VIX from CBOE CDN + SPY/QQQ/DIA from Moomoo TCP
  • getSectorPerformance() → 11 sector ETFs (Moomoo batch → Yahoo fallback per miss)
  • getMarketMovers() → top day gainers/decliners/most-active (Yahoo Finance)
  • getEarningsCalendar() → today's earnings reporters (Yahoo Finance)

Outputs: regime (trending / volatile / news-driven / choppy) · direction (bullish / bearish / neutral) · leading sectors · today's catalysts
If regime = "choppy" → abort scan entirely — no clear direction
Moomoo TCP (indices/sectors) Yahoo Finance (movers/earnings) CBOE CDN (VIX)
4
buildCandidates() — pick who to analyze (pure logic, zero API calls)
Uses only data from Step 3 — no new network requests:
  • Watchlist symbols — always included (user's custom list + default 20 large-caps)
  • Day gainers — added if regime = trending or news-driven
  • Day decliners — added if regime = volatile (bounce plays near support)
  • Earnings reporters get +3 weight boost if regime is news-driven
  • Stocks in leading sector get +1 weight boost
Results: up to 12 candidates sorted by relevance weight (most likely to qualify scores highest)
Pure deterministic logic — zero latency Reuses Step 3 data
5
Score all 12 candidates in parallel
Promise.allSettled(candidates.map(sym => getConvictionScore(sym)))
All 12 scored simultaneously — total time typically 3–8 seconds
Each score call fetches in parallel internally:
  • Real-time price + RVOL + ATR (Moomoo TCP)
  • 90-day OHLCV for RSI/MACD/EMA/BB computation (Moomoo → Yahoo fallback)
  • Recent news < 48h (Yahoo Finance / FinancialJuice)
  • Analyst ratings + price target (Yahoo Finance, 4-hour cache)
  • Insider buying Form 4 (OpenInsider scrape)
Failed fetches score zero for that factor — never crash the whole scan
Moomoo TCP (real-time) Yahoo Finance (news/ratings) TradingView CDP (optional)
6
selectBestTrade() — filter and pick the single winner
All 12 scored candidates filtered through gates in order:
☑ Score ≥ min_conviction (your threshold, default 50/100)
☑ No existing open position in this symbol
☑ Symbol did NOT stop out in last 60 minutes (re-entry block)
☑ Sector not in your sectors_blocklist
☑ Historical win rate for current regime is acceptable
→ Returns the single highest-scoring qualifying candidate, or null
Deterministic filter engine PostgreSQL trade history (re-entry check)
7a
No qualifying trade
Score below threshold
Logged to scan history with top candidates + scores
Wait 10 minutes for next cycle
Telegram ping if manually triggered
No action taken
7b
Execute the trade
Position size = target_profit ÷ (ATR × target_mult)
Stop = entry − (ATR × 1.5×)
Target = entry + (ATR × 3.0×)
placeMoomooTrade() → Exchange
saveDailyPick() → PostgreSQL
Telegram: symbol · entry · stop · target · score
Moomoo → Exchange PostgreSQL daily_picks
🎯 How the 0–100 Conviction Score is Built
Six independent factors scored in parallel and combined into a single number. A stock needs ≥ 50 to qualify for a trade. Here's exactly what goes into each factor.
Score composition

News 25
Momo 20
Anlst 10

5
0= 100 pts max
30 pts
Technical Analysis
TV CDP → Moomoo 90-day OHLCV → Yahoo Finance (cascade fallback)
📈 RSI in 40–65 sweet spot — not overbought (>70), not oversold (<30)
📊 MACD histogram positive and rising — bullish momentum confirmed
🔼 Price above both EMA20 and EMA50 — aligned with medium + short-term trend
🎯 Bollinger Band %B position 0.4–0.7 — midrange, room to run without overextension
Fallback chain: TradingView live indicators → Moomoo 90-day OHLCV (compute indicators in JS) → Yahoo Finance daily OHLCV (compute indicators in JS)
25 pts
News Catalyst
FinancialJuice (< 4h) · Yahoo Finance news · Alpaca news API
📰 Fresh news within 48 hours = full points. Older = scaled down. No news = zero
⚡ Breaking news < 4h from FinancialJuice gets highest weight — closest to market reaction
💰 Earnings surprise magnitude: big beat boosts score, miss penalizes
📅 Pre-earnings drift: does this stock historically run up before reporting? If yes, +points
Staleness rule: News > 48h is flagged as stale and not traded on. Macro news (tariffs, Fed) is mapped to beneficiary/victim tickers rather than the macro topic itself.
20 pts
Momentum
Moomoo TCP real-time → Yahoo Finance daily (fallback)
🚀 RVOL > 1.5× 21-day average volume — unusual activity is the market "voting" with money
📈 Price change today > 0% — stock is already moving in the right direction
💪 Volume confirmation — price move must be backed by above-average volume
📉 ATR (Average True Range) computed here — used later for stop/target sizing
RVOL formula: today's volume ÷ 20-day average volume. RVOL > 2.0× = strong signal. > 3.0× = exceptional. Moomoo is the primary source; Yahoo 30-day OHLCV used as fallback.
10 pts
Analyst Consensus
Yahoo Finance quoteSummary · 4-hour cache per symbol
⭐ Mean analyst rating < 2.5 (scale: 1.0=Strong Buy · 2.0=Buy · 3.0=Hold · 5.0=Strong Sell)
🎯 Price target upside > 5% from current price (analyst consensus target vs. live price)
👥 Buy + Strong Buy count vs. Sell + Strong Sell count — net buy recommendation
Cache strategy: Analyst ratings change slowly (weekly/monthly), so 4-hour cache is safe and avoids hammering Yahoo. Uses yahoo-finance2 npm package which auto-handles auth tokens.
10 pts
Market Conditions
Reuses Step 3 context data — no additional API call
📊 VIX level: <18 = full points · 18–25 = partial · >25 = reduced · >35 (crisis) = zero
🎯 Regime match: stock is trending in line with the overall market regime
🏆 Sector leadership: stock's sector is in the current "leading sectors" list from context
Cost = $0: Reuses data already fetched in getMarketContext(). No extra API calls, no extra latency.
5 pts
Insider Activity
OpenInsider.com · SEC Form 4 filings
🏛️ Executive purchases (Form 4 SEC filings) in the last 30 days
📊 Net buying volume > selling volume = positive signal
🎯 Cluster buying (multiple insiders buying simultaneously) = maximum 5 points
Failure handling: If OpenInsider is unavailable, this factor scores zero and the other 95 points still apply normally.

📡 Data Sources & Fallback Chains
Every external integration — what it provides, how it's used, and what happens when it fails.
PRIMARY
🟢 Moomoo OpenD
TCP socket · 127.0.0.1:11111
Provides: Real-time bid/ask/last price · OHLCV K-lines (daily + intraday) · Portfolio positions with live P&L · Open orders · Cash balance & buying power · ATR computation · RVOL vs 20-day avg · Trade execution (paper + live)

Mode: MOOMOO_TRADE_ENV=SIMULATE (paper trading) or REAL (live money). OpenD must be running and logged in before server starts. TCP connection is persistent — reconnects automatically on drop.
FALLBACK
🟡 Yahoo Finance
HTTPS · query2.finance.yahoo.com · No API key
Provides: Historical OHLCV 90-day daily · Analyst ratings + consensus price targets · Earnings calendar + historical surprise data · Symbol news (7 days) · Market movers (gainers/decliners/most active) · Futures prices (ES=F, NQ=F) · Short interest · ^VIX fallback
Fallback for: Moomoo when OpenD is unavailable. Always used for: futures (Moomoo can't query ES=F), analyst ratings, historical earnings, insider-mapped news.
No API key required — uses unofficial public endpoint. Random User-Agent string to avoid rate limiting. Analyst data cached 4 hours. News cached 15 min. Timeout: 6–7 seconds per request.
FREE
⚡ CBOE VIX CDN
HTTPS · cdn.cboe.com/api/global/delayed_quotes/quotes/_VIX.json
Provides: VIX index current level + daily change · ~15-second delay · No API key · No authentication · No rate limits
Fallback: Yahoo Finance ^VIX (if CBOE CDN returns non-200)
Primary VIX source. Polled in getMarketSentiment() every 2 minutes (with TTL cache). VIX level is the single most important input to market regime classification and position sizing decisions.
OPTIONAL
📊 TradingView Desktop
Chrome DevTools Protocol · localhost:9222
Provides: Live indicator values from your actual chart (RSI, MACD histogram, EMA20/50, Bollinger Bands) · Custom Pine indicator drawn levels (support/resistance lines) · Labeled price levels from scripts · Current chart symbol + timeframe
Fallback when TV is off: Moomoo 90-day OHLCV → RSI/MACD/EMA/BB computed in pure JavaScript (same math, slightly delayed data)
Availability checked every 15 seconds (reduced from 60s to detect recovery faster). System degrades gracefully — no crash, no error to user. Requires TradingView launched with --remote-debugging-port=9222 flag.
